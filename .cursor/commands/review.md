对当前分支的变更进行代码审查。

1. 运行 `git diff main...HEAD`（或对应的基础分支）查看所有变更
2. 对每个变更文件进行检查：
   a. **规格符合性**：变更是否与 commit message / PR 描述一致？
   b. **代码规范**：命名、结构、注释是否符合项目约定？
   c. **安全检查**：输入验证、权限控制、敏感信息泄露
   d. **错误处理**：边界情况、异常路径
   e. **测试覆盖**：关键路径是否有测试？
3. 如果项目有 linter / type checker，运行并报告结果
4. 输出结构化审查报告（参照 Kyle 的 Stage 1 + Stage 2 格式）
5. 将审查报告保存到 `.dev-agents/shared/reviews/`

注意事项：
- 以只读模式审查，不修改任何代码
- 问题按严重度分级：严重 / 重要 / 建议
- 必须附带具体文件路径和行号
