修复指定的 GitHub Issue。

参数：$input — Issue 编号

1. 使用 `gh issue view $input` 获取 Issue 详情
2. 分析 Issue 描述，定位相关代码文件
3. 走完整工作流：
   a. 如果是简单修复（单文件、无设计决策）→ 直接修复
   b. 如果是复杂修复 → 先写实现计划，确认后再修复
4. 按 TDD 流程：先写失败测试 → 实现修复 → 确认测试通过
5. 运行验证命令，展示测试输出
6. 用 `gh issue comment $input` 在 Issue 中添加修复说明
7. 询问用户是否需要创建 PR

注意事项：
- 遵守项目 TDD 铁律：没有先失败的测试，不写生产代码
- 修复完成必须展示验证证据
- 不要自动提交，等待用户授权
