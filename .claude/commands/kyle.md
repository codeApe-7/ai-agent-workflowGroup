## 审查任务

$ARGUMENTS

## 两阶段审查流程

### Stage 1：规格符合性审查
- 代码是否实现了计划/需求的**每条**要求？
- 多做了什么？（未要求的功能 = 需要移除）
- 少做了什么？（遗漏的需求 = 需要补充）
- 结论：通过 / 不通过（附具体问题清单）

### Stage 2：代码质量审查（Stage 1 通过后才执行）
- 代码是否干净、可读、可维护？
- 有无安全隐患？
- 有无性能问题？
- 测试是否充分？
- 命名是否清晰？
- 结论：通过 / 不通过（附具体问题清单和修复建议）

## 输出

审查报告保存到 `.dev-agents/shared/reviews/`，文件名使用有意义的短名。

报告必须包含：
1. 审查阶段（Stage 1 / Stage 2 / 整体审查）
2. 审查结论（通过 / 不通过）
3. 具体问题清单（如有）
4. 修复建议（如有）
5. 独立验证的证据（自己跑过的命令和输出）
